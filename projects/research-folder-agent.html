05 · EVALUATION

自动评估检查工程结果，人工复核判断科研内容

自动评分用于比较任务完成度和输出规范，不被解释为“科学准确率”；最终科研结论仍必须由研究者本人核验。

自动评估标准

1
任务完成summary 是否存在并包含有效内容
2
模型调用增强模式下真实 API 是否完成重写
3
输出规范生成文件是否符合预设格式和目录规则
4
文件覆盖源文件数量、支持文件数量与处理比例
5
来源追踪日报中的关键内容能否追溯到源文件
输出：PASS / CHECK、综合评分、文件统计、来源追踪率与 API 状态。

仍需人工科研判断

!
科学准确性关键数据和计算结果是否被正确理解
!
结论层级是否把初步观察误写成确定结论
!
重要信息是否遗漏异常样本、方法限制或关键机理
!
后续可用性总结是否足以支持汇报、写作和下一轮计算决策
工具负责整理和提示，方向、标准与最终科研判断仍由研究者本人把关。
06 · REFLECTION

工作流已经验证，下一步是保留科研文件中的领域语义

第一版证明了完整流程可行，也暴露出更关键的问题：科研文件不同于普通办公文档，能够读取文件，不等于能够理解其中的科研含义。

KEY REFLECTION

通用文档解析解决了格式统一，但无法自动补足计算化学语义。

Excel、PDF、Notebook 和普通文本可以较稳定地转换为 Markdown；但 VASP、Bader、LOBSTER、结构文件和科研图像包含专门字段、数值关系和物理含义，需要结合任务本身设计定制化解析规则。

测试暴露的领域缺口

大文件夹和科研专用文件中，通用转换会削弱信息完整性和来源追踪。

  • 专用文件未被充分解析
  • 图像和结构文件缺少语义理解
  • 一次性压缩导致部分来源被省略
  • 科学结论仍无法仅靠自动评分判断

下一阶段的设计响应

针对真实缺口重构解析和总结流程。

  • 增加 VASP / Bader / LOBSTER 专用 parser
  • 按 20–30 个文件分块，再层级汇总
  • 为关键结论生成独立来源索引
  • 建立人工校验集，检查遗漏和过度解释
